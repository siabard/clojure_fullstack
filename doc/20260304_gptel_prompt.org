*** 설정 

You are an expert Clojure and ClojureScript full-stack developer with deep knowledge of functional programming, REPL-driven development, and Lisp paradigms. 

Your core tech stack includes:
- Backend: Clojure, tools.deps (deps.edn), Ring, Reitit for REST APIs.
- Frontend: ClojureScript, shadow-cljs, Reagent, re-frame.
- JS Interop: Seamless integration of npm packages (e.g., luxon, react).

When writing code or providing guidance, strictly adhere to the following rules:
1. Write Idiomatic Code: Emphasize immutability, pure functions, destructuring, and proper use of threading macros (`->`, `->>`).
2. Architecture & Separation of Concerns: In re-frame, strictly separate `events`, `subs`, and `views`. Keep business logic out of views.
3. JS Interop: Use modern shadow-cljs import syntax for npm packages (e.g., `["luxon" :refer [DateTime]]`). Prefer `.-property` and `.method` syntax for JS object manipulation.
4. Step-by-Step Chunking: We are building incrementally. Provide modular, bite-sized code chunks rather than overwhelming monolithic blocks. Do not rewrite unchanged parts of a file.
5. Emacs/REPL Context: Assume the user is evaluating your code in an Emacs REPL (CIDER). Ensure code snippets are self-contained and easy to evaluate.
6. Communication: Be concise, direct, and focus on the code and architectural decisions. Avoid unnecessary boilerplate explanations.


*** 수립전략


Emacs에서 =gptel-agent= 를 활용한 Clojure/ClojureScript 풀스택 개발, 아주 훌륭한 접근입니다. REPL 기반의 Lisp 환경과 AI 에이전트의 결합은 생산성을 극대화할 수 있는 강력한 조합입니다.

전체 개발 계획을 잡고 단계별로 진행하기 위해서는, 에이전트에게 **명확한 역할(Persona)**을 부여하고 **작업을 작은 단위로 쪼개어(Chunking)** 지시하는 것이 핵심입니다.

**** 1. gptel-agent 초기 설정 및 계획 수립 전략

=gptel-agent= 를 효과적으로 사용하려면 처음부터 코드를 짜라고 지시하기보다, 프로젝트의 전체적인 청사진과 컨벤션을 먼저 합의하는 과정이 필요합니다.



- **컨텍스트 고정:** 기술 스택(Clojure, re-frame, REST 등)과 프로젝트 디렉토리 구조를 에이전트에게 명확히 인지시킵니다.
- **파일 단위의 지시:** 에이전트가 한 번에 너무 많은 파일을 수정하게 하면 환각(Hallucination)이 발생하거나 코드가 꼬일 수 있습니다. 프론트엔드와 백엔드, 설정 파일을 엄격히 분리하여 단계별로 지시해야 합니다.
- **JS Interop 명시:** React 생태계나 Luxon 같은 JS 라이브러리를 사용할 것임을 미리 알려, `shadow-cljs` 설정이나 namespace 선언 시 `["luxon" :as DateTime]` 형식의 interop 문법을 올바르게 작성하도록 유도합니다.

**** 2. 단계별 구체적 지시사항 (Prompts)

Emacs 버퍼에서 `gptel`을 열고 다음 프롬프트들을 단계별로 입력하여 작업을 진행해 보세요.

### 단계 0: 시스템 프롬프트 및 프로젝트 뼈대 기획 (Plan)

가장 먼저 에이전트에게 프로젝트의 전체 구조를 제안하도록 지시합니다.

> **Prompt 0:**
> "너는 경험이 풍부한 Clojure/ClojureScript 풀스택 개발자야. 우리는 지금부터 간단한 Todo 애플리케이션을 만들 거야.
> **기술 스택:**
> * Backend: Clojure, Ring/Reitit (REST API 구축)
> * Frontend: ClojureScript, re-frame, Reagent
> * Build Tool: tools.deps (`deps.edn`), shadow-cljs
> * JS Interop: 프론트엔드에서 Luxon(날짜 처리) 등 npm 패키지 사용
> 
> 
> 코드를 바로 작성하지 말고, 먼저 프로젝트의 디렉토리 구조와 `deps.edn`, `shadow-cljs.edn`에 들어갈 핵심 의존성 목록을 정리해서 보여줘. 프론트엔드와 백엔드 코드가 분리된 모노레포 형태의 디렉토리 구조를 제안해 줘."

### 단계 1: 설정 파일 및 빌드 환경 구성

에이전트가 제안한 구조를 확인한 후, 실제 설정 파일을 작성하도록 지시합니다.

> **Prompt 1:**
> "좋아, 제안해 준 디렉토리 구조를 바탕으로 프로젝트 루트에 들어갈 `deps.edn`과 `shadow-cljs.edn` 파일의 전체 코드를 작성해 줘.
> `shadow-cljs.edn`에는 `:app` 빌드 타겟을 설정하고, `package.json`에 `luxon`과 `react`, `react-dom`을 포함시키는 방법도 함께 설명해 줘."

### 단계 2: 백엔드 REST API 구현

데이터베이스는 복잡성을 줄이기 위해 일단 Clojure의 `atom`을 사용하도록 지시합니다.

> **Prompt 2:**
> "이제 백엔드 REST API를 작성하자. `src/backend/core.clj` (또는 네가 제안한 백엔드 엔트리포인트) 파일에 다음 기능을 구현해 줘.
> * 메모리 기반 상태 관리 (`atom` 사용)
> * Reitit과 Ring을 사용한 라우팅
> * API 엔드포인트:
> * `GET /api/todos` (목록 조회)
> * `POST /api/todos` (생성)
> * `PUT /api/todos/:id` (수정/완료 처리)
> * `DELETE /api/todos/:id` (삭제)
> 
> 
> * JSON 요청/응답 처리를 위해 `muuntaja` 미들웨어를 적용해 줘."
> 
> 

### 단계 3: 프론트엔드 - Re-frame Event & Subscriptions

프론트엔드의 핵심 비즈니스 로직과 상태 관리를 먼저 구성합니다.

> **Prompt 3:**
> "백엔드가 완성되었으니 프론트엔드 상태 관리를 설정하자. re-frame을 사용하여 `events.cljs`와 `subs.cljs` 파일을 작성해 줘.
> * `events.cljs`:
> * 초기 상태 설정 (`:initialize-db`)
> * 백엔드 API와 통신하기 위한 HTTP 요청 이벤트 (`day8.re-frame.http-fx` 활용 권장)
> * Todo 추가, 토글, 삭제 이벤트
> 
> 
> * `subs.cljs`:
> * 전체 Todo 목록을 가져오는 subscription"
> 
> 
> 
> 

### 단계 4: 프론트엔드 - UI Component & JS Interop

실제 뷰를 그리고 npm 패키지를 연동하는 단계입니다.

> **Prompt 4:**
> "마지막으로 Reagent를 사용해 UI 뷰(`views.cljs`)를 작성해 줘.
> **요구사항:**
> 1. Todo 입력 폼과 Todo 리스트를 렌더링해야 해.
> 2. Javascript Interop을 적극적으로 보여주기 위해, 각 Todo 아이템이 생성된 시간을 표시할 때 `luxon` 라이브러리를 import해서 날짜를 포맷팅하는 로직을 컴포넌트 내에 포함시켜 줘. (예: `["luxon" :refer [DateTime]]` 사용)
> 3. `core.cljs`에 앱을 DOM에 마운트하는 코드도 함께 제공해 줘."
